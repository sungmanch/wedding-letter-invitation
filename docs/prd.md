## Executive Summary

청모장은 결혼을 앞둔 예비 신부가 청첩장 모임(청모)을 쉽게 준비할 수 있도록 돕는 올인원 서비스입니다.

**해결하려는 문제:**
예비 신부는 결혼 준비로 바쁜 와중에도 여러 그룹의 청첩장 모임을 준비해야 합니다. 각 그룹마다 일정 조율, 메뉴 선정, 장소 선정, 식당 예약을 반복해야 하며, 이는 큰 심적 부담이 됩니다.

**현재 대안의 문제점:**
- 메신저로 일정 조율
- 구글 폼으로 취향 조사
- 네이버/카카오맵으로 식당 검색
- 전화로 식당 예약

→ 모든 과정이 수작업이며, 여러 서비스를 오가야 합니다.

**청모장의 솔루션:**
URL 하나를 그룹에 공유하면, 구성원들의 취향을 자동 수집하고 AI가 최적의 식당을 추천합니다. 추가로 친구들의 축하 편지를 모아 단순한 모임이 아닌 소중한 추억으로 만들어 줍니다.

### What Makes This Special

**1. 원스톱 통합 경험**
메신저 + 설문 + 식당 검색 + 예약 안내를 하나의 서비스로 통합

**2. 평범한 모임을 추억으로**
단순한 식당 선정을 넘어, 친구들의 진심 어린 편지를 모아 평생 간직할 추억을 선물

**3. Host의 부담 최소화**
URL 하나 공유 → 자동 취향 수집 → AI 추천 → 결과 공유까지 최소한의 노력으로 완료

## Success Criteria

### Primary Success Metrics (2주 기준)

| 지표 | 목표 | 측정 방법 |
|------|------|----------|
| 회원가입 | 100명 | SEO 오가닉 유입 |
| 설문 생성 | 30명 (30% 전환) | 청모장 생성 완료 수 |
| 편지 열람 경험 | 6명 (20% of 30) | 편지 열람 페이지 도달 |

### Success Definition

**MVP 성공 = 6명의 Host가 친구들의 편지를 열람하는 경험을 완료**

이는 전체 사용자 여정(청모장 생성 → 설문 공유 → 응답 수집 → 식당 추천 → 편지 열람)이 정상 작동함을 의미합니다.

### Acquisition Strategy

- **채널:** SEO 기반 오가닉 유입
- **타겟 키워드:** 청첩장 모임, 결혼 전 친구 모임, 신부 친구 모임 식당
- **콘텐츠:** 청모 준비 가이드, 식당 추천 팁

## Product Scope

### MVP - Minimum Viable Product

청모장의 핵심 가치를 검증하기 위한 최소 기능 세트

**Core Flow:**
1. **청모장 생성** - Host가 그룹명, 선호 옵션 입력
2. **설문 링크 공유** - 카카오톡/링크 복사로 친구들에게 전달
3. **설문 응답** - Guest가 회원가입 없이 취향 응답 + 편지 작성
4. **AI 식당 추천** - 수집된 응답 기반 Top 3-5 식당 추천
5. **편지 열람** - 2주 대기(무료) 또는 결제(즉시) 후 카드뷰 열람
6. **청모장 공유** - 선택한 식당과 일정을 템플릿으로 친구들에게 공유

**MVP 포함 기능:**
- 청모장 CRUD
- 고유 설문 URL 생성
- 카카오톡 공유
- 설문 응답 (음식 종류, 가격대, 분위기, 식이 제한)
- 편지 작성 (300자 + 이모지/스티커)
- 응답 현황 대시보드
- Rule-based 식당 추천 알고리즘
- 2주 대기 / 계좌이체 결제 편지 열람
- 운영자 수동 열람 권한 부여 (Admin)
- 1종 청모장 템플릿
- 모바일 first 반응형 웹

### Growth Features (Post-MVP)

**1. 수익화 고도화**
- 결제 시스템 자동화 (PG 연동)
- 프리미엄 템플릿 (유료)
- 추가 스티커/이모지 팩

**2. 사용자 경험 개선**
- 청모장 템플릿 다양화 (4-5종)
- 편지 열람 애니메이션 효과
- 알림톡/문자 리마인더

**3. 기능 확장**
- 식당 예약 연동 (캐치테이블, 네이버 예약)
- 참석자 일정 투표 기능
- 사진 첨부 편지

### Vision (Future)

**청모장 → 모임장**
- 결혼 외 다양한 모임으로 확장 (생일, 환송회, 동창회)
- B2B 기업 단체 모임 서비스
- 글로벌 확장 (해외 한인 커뮤니티)

## Functional Requirements

### FR-1: 청모장 생성 및 관리

| ID | 요구사항 |
|----|----------|
| FR-1.1 | Host는 고유한 청모장을 생성할 수 있다 |
| FR-1.2 | Host는 청모장에 그룹 이름을 설정할 수 있다 |
| FR-1.3 | Host는 선호 옵션(예상 인원, 지역, 예산)을 미리 설정할 수 있다 |
| FR-1.4 | Host는 자신이 만든 청모장 목록을 조회할 수 있다 |
| FR-1.5 | Host는 청모장을 수정하거나 삭제할 수 있다 |

### FR-2: 설문 링크 공유

| ID | 요구사항 |
|----|----------|
| FR-2.1 | 시스템은 청모장별 고유한 설문 URL을 생성한다 |
| FR-2.2 | Host는 설문 링크를 클립보드에 복사할 수 있다 |
| FR-2.3 | Host는 카카오톡으로 설문 링크를 공유할 수 있다 |

### FR-3: 응답 현황 관리

| ID | 요구사항 |
|----|----------|
| FR-3.1 | Host는 실시간 응답 현황을 확인할 수 있다 |
| FR-3.2 | Host는 응답자와 미응답자를 구분하여 볼 수 있다 |
| FR-3.3 | 설문 완료 시 운영자에게 알람이 전달된다 |

### FR-4: 설문 응답

| ID | 요구사항 |
|----|----------|
| FR-4.1 | Guest는 회원가입 없이 설문에 응답할 수 있다 |
| FR-4.2 | 시스템은 필수 질문 응답 여부를 검증한다 |
| FR-4.3 | Guest는 이름을 필수로 입력해야 한다 |
| FR-4.4 | Guest는 제출 후 응답을 수정할 수 없다 |

### FR-5: 편지 작성

| ID | 요구사항 |
|----|----------|
| FR-5.1 | Guest는 설문과 함께 축하 편지를 작성할 수 있다 (선택) |
| FR-5.2 | 편지는 300자 이내로 제한된다 |
| FR-5.3 | Guest는 편지에 이모지/스티커를 추가할 수 있다 |

### FR-6: AI 식당 추천

| ID | 요구사항 |
|----|----------|
| FR-6.1 | 시스템은 응답을 종합하여 매칭 점수를 산출한다 |
| FR-6.2 | 시스템은 Top 3-5개 식당을 추천한다 |
| FR-6.3 | 각 식당의 매칭 이유가 설명된다 |
| FR-6.4 | 식당 추천 완료 시 Host에게 이메일이 발송된다 |

### FR-7: 편지 열람

| ID | 요구사항 |
|----|----------|
| FR-7.1 | Host는 청모장 생성 2주 후 무료로 편지를 열람할 수 있다 |
| FR-7.2 | 편지는 카드뷰 형태로 표시된다 |
| FR-7.3 | Host는 결제(계좌이체) 후 즉시 편지를 열람할 수 있다 |
| FR-7.4 | 운영자는 수동으로 열람 권한을 부여할 수 있다 |
| FR-7.5 | 한 번 열람 권한을 얻으면 영구적으로 열람 가능하다 |

### FR-8: 청모장 공유

| ID | 요구사항 |
|----|----------|
| FR-8.1 | Host는 선택한 식당과 모임 일정을 입력할 수 있다 |
| FR-8.2 | 청모장은 1종의 디자인 템플릿으로 생성된다 |
| FR-8.3 | Host는 청모장 미리보기를 확인할 수 있다 |
| FR-8.4 | Host는 카카오톡/링크로 청모장을 공유할 수 있다 |
| FR-8.5 | 청모장은 모바일 first 웹 페이지로 표시된다 |

## User Experience Principles

### Visual Identity

- **Primary Color:** Blush Pink (#FFB6C1)
- **Accent Color:** Soft Gold (#D4AF37)
- **Tone:** 따뜻하고 로맨틱한 웨딩 테마
- **Typography:** 깔끔하고 읽기 쉬운 한글 폰트

### Interaction Principles

- **모바일 First:** 터치 친화적 UI, 큰 버튼, 스와이프 제스처
- **최소 입력:** 선택형 질문 위주, 타이핑 최소화
- **진행 상태 표시:** 프로그레스 바로 현재 단계 안내
- **즉각적 피드백:** 버튼 클릭, 제출 완료 시 시각적 피드백

### Key User Flows

**Host Flow:**
```
랜딩 → 청모장 만들기 → 기본 정보 입력 → 설문 링크 공유 → 응답 대기
→ 식당 추천 확인 → 식당 선택 → 편지 열람 → 청모장 공유
```

**Guest Flow:**
```
링크 클릭 → 이름 입력 → 설문 응답 (3분) → 편지 작성 (선택) → 제출 완료
```
